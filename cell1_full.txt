    <b>1. Integer (<code>int</code>) : <span style="color: #070707ff; font-size: 0.85em; font-weight: normal;">Precision Counts & Identifiers</span></b>
</h2>

<div style="border-left: 4px solid #4DB6AC; padding-left: 15px; margin: 15px 0; color: #CCCCCC;">
    <b>≡ƒöì What is it?</b><br>
    An integer is a whole number that can be positive, negative, or zero, and it does not contain a decimal point. Unlike many other programming languages such as C or Java, Python integers support <b>arbitrary precision</b>, meaning they can grow to very large values limited only by your system's memory.
</div>

**Examples:** `10`, `-25`, `0`, `999999`

```python
x = 10**100  # A 101-digit number!
print(x)
```

<h3 style="color: #64B5F6; background-color: #0F1A2A; padding: 8px 12px; border-radius: 4px; margin-top: 25px;"><b>≡ƒÆ╝ Why Data Analysts Care</b></h3>

Integers are the foundational blocks of tracking entities and loops.

* **Row Counts**: Functions like `len(df)` return integers.
* **Primary Keys**: `Customer ID`, `Order ID`, and `Transaction ID` are typically stored as integers for high-speed indexing.
* **Categorical Encoding**: Converting string categories to integers (`0 = Low`, `1 = Medium`, `2 = High`) is a standard requirement for Machine Learning.

<h3 style="color: #FFB74D; background-color: #2B1A0D; padding: 8px 12px; border-radius: 4px; margin-top: 25px;"><b>ΓÜá∩╕Å Common Pitfall: Division Behavior</b></h3>

<div style="border-left: 4px solid #FFB74D; padding-left: 15px; margin: 15px 0; color: #CCCCCC;">
    In Python, the standard division operator <code>/</code> <b>always</b> returns a float, even if the result is mathematically whole.
</div>

```python
print(10 / 2)   # Output: 5.0 (float)
print(10 // 2)  # Output: 5 (int) - Use floor division to keep it an integer!
```
> ΓÜí **Important**: Floor division truncates toward negative infinity (e.g., `-7 // 2` yields `-4`).

<h3 style="color: #81C784; background-color: #102414; padding: 8px 12px; border-radius: 4px; margin-top: 25px;"><b>≡ƒºá Memory Insight</b></h3>

<div style="border-left: 4px solid #81C784; padding-left: 15px; margin: 15px 0; color: #CCCCCC;">
    A standard Python integer uses about <b>28 bytes</b>. For large-scale data processing (millions of rows), this is highly inefficient. Professional analysts use NumPy and pandas optimized types to compress data footprint.
</div>

| Optimized Type | Memory Usage | Range |
| :--- | :--- | :--- |
| `int8` | 1 byte | -128 to 127 |
| `int16` | 2 bytes | -32K to 32K |
| `int32` | 4 bytes | ~┬▒2 billion |
