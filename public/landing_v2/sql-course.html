The Complete 18-Day SQL Curriculum

Every concept a professional Data Analyst uses on the job — in a logical, progressive sequence.

Phase 1 Foundations — Days 1–4 📦 retail.db
  • Day 01
    Database Fundamentals & SQL Environment
    Relational databases vs spreadsheets, schemas, tables, fields, SELECT *, SELECT columns, aliases (AS), DISTINCT, LIMIT
    🎯 Schema Explorer: Fetch unique product categories sold
  • Day 02
    SELECT & Filtering — Retrieving Rows
    WHERE clause filtering, logical operators (AND, OR, NOT), operators (=, <>, <, >, <=, >=), BETWEEN, IN list filters
    🎯 High-Value Filter: Find transactions over ₹8,000 in the 'East' region
  • Day 03
    Sorting, Pattern Matching & CASE
    ORDER BY sorting (ASC/DESC, multi-column), LIKE wildcard fuzzy match (% and _), NULL comparisons (IS NULL, IS NOT NULL), COALESCE fallback, CASE WHEN conditional logic
    🎯 Customer Clean-up: Sort 'Kumar' occurrences by transaction amounts; handle null email defaults
  • Day 04
    Aggregate Functions & GROUP BY
    Aggregate metrics (COUNT, SUM, AVG, MIN, MAX), GROUP BY groupings, HAVING post-aggregation filter vs WHERE pre-filter
    🎯 Category Volume: Summarize average cost and order counts; extract high-volume cohorts
Phase 2 Joins & Advanced Filtering — Days 5–7 📦 company.db / ecommerce.db
  • Day 05
    Joins — Combining Multiple Tables
    Primary/foreign keys, INNER JOIN, LEFT JOIN, SELF JOIN, CROSS JOIN, table aliasing, multiple table joins
    🎯 Department Heads: Retrieve active departments with their assigned managers and budgets
  • Day 06
    Subqueries
    Scalar subqueries, multi-row subqueries (IN, NOT IN), correlated subqueries, EXISTS/NOT EXISTS vs JOIN performance
    🎯 Spender Cohort: Retrieve orders representing spending values above the cohort's average
  • Day 07
    Common Table Expressions (CTEs)
    WITH clause syntax, readability improvements, chaining multiple CTEs, recursive CTE basics, CTEs vs derived tables
    🎯 Category Revenue: Chaining CTEs to organize and output product sales totals by category
Phase 3 Window Functions & Analytics — Days 8–10 📦 ecommerce.db
  • Day 08
    Window Functions I — Ranking & Numbering
    OVER(), PARTITION BY, ROW_NUMBER(), RANK(), DENSE_RANK() ties, NTILE(n) distribution quartiles
    🎯 Region Rankings: Rank customers by transactions within each region without collapsing rows
  • Day 09
    Window Functions II — Analytics & Frames
    LAG(), LEAD() chronological offset tracking, running totals, moving average window framing (ROWS BETWEEN)
    🎯 Cumulative Spend: Construct running aggregates of customer orders to trace user lifetime value
  • Day 10
    Date & Time Functions
    ISO date storage formats, STRFTIME extraction, date math, JULIANDAY intervals, weekend date analysis
    🎯 Signup Intervals: Calculate the average day latency between account registration and first purchase
Phase 4 Data Cleaning & Optimization — Days 11–14 📦 ecommerce.db
  • Day 11
    String & Type Functions
    Text manipulation (UPPER, LOWER, TRIM, REPLACE, SUBSTR, INSTR), CAST conversions, null fallbacks, NULLIF division safety
    🎯 Domain Extraction: Isolate and standardize domain patterns from messy registration emails
  • Day 12
    Data Cleaning & Wrangling in SQL
    Deduplication using ROW_NUMBER(), data profiling, outlier detection, pivoting rows to columns (the pivot trick)
    🎯 Status Pivot: Transform and report purchase metrics pivoted across regional sales channels
  • Day 13
    Set Operations & Advanced Joins
    UNION, UNION ALL duplication performance, INTERSECT overlaps, EXCEPT exclusions, anti-joins, semi-joins
    🎯 Active Spenders: Intersect Q3 and Q4 buyers; extract list of active dormant accounts
  • Day 14
    Query Optimisation & Execution Plans
    EXPLAIN QUERY PLAN analysis, table scans vs index search, sargable predicates, clustered vs non-clustered indexes
    🎯 Index Optimization: Analyze table scans and verify index efficiency for subquery filters
Phase 5 Reusability & Advanced Analytics — Days 15–17 📦 company.db / ecommerce.db
  • Day 15
    Views, Stored Procedures & Functions
    Virtual tables (CREATE VIEW), security layers, stored procedures transactions, database triggers and audits
    🎯 Management View: Build a reusable view mapping staff members to managers and budgets
  • Day 16
    Advanced Analytics Patterns
    Cohort analysis retention grids, marketing funnel conversion ratios, RFM scoring segmentation profiles
    🎯 RFM Segments: Partition customers into monetary and recency brackets using NTILE
  • Day 17
    Data Modelling & Analytics Engineering
    Dimensional schema design (fact/dimension tables), star vs snowflake structures, dbt pipelines, SCD Type 1/2 history
    🎯 Schema Design: Inspect foreign relationships and model a grain-specific intermediate mart layer
Capstone BI Capstone & Interview Prep — Day 18 📦 capstone_retail.db
  • Day 18
    🏆 SQL Interview Prep & Real-World Patterns
    Capstone Business Intelligence Report: write production-ready SQL queries answering senior analytical questions. Top-N rankings, MoM revenue growth, cohort spend intervals, returns and churn rates.
    🎯 Capstone Project: Output a clean, fully-optimized SQL report detailing retail performance KPIs
